# AI.Web Slice 16 Selected Meaning Boundary Scaffold

Slice 16 is a boundary-only scaffold.

It defines how one candidate meaning can be represented as selected without allowing that selected meaning to become a later authority category.

## Core laws

- Selected meaning is not truth.
- Selected meaning is not permission.
- Selected meaning is not delivery.
- Selected meaning is not execution.
- Selected meaning is not final meaning selection.
- Selected meaning is not evidence validation.
- Selected meaning is not memory authority.
- Selected meaning is not tool routing.
- Selected meaning is not output rendering.

## Accepted representation scope

The scaffold may represent candidate selection references, selection basis records, selection constraints, selected-meaning status records, selection trace records, non-selected candidate references, confidence and uncertainty boundaries, blocked or refused selection boundaries, and selection receipts.

## Explicit non-authority

This scaffold does not decide truth, grant permission, authorize action, invoke tools, deliver output, execute anything, write memory, validate evidence, admit external resources, render output, or perform final meaning selection.

## Prior boundary dependency

Slice 16 depends on the existence of prior candidate, ambiguity, concept, predicate, gate, external-resource, and corpus/evidence/memory/trace boundaries. It does not bypass them.
